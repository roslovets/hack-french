# Hack French 🕵️‍♂️🇫🇷

> Не зубри французский. Взламывай, как он работает.

**Hack French** — игра-лаборатория для reverse engineering французского языка. Вместо скучных
уроков пользователь проходит языковые расследования: видит странную французскую фразу, собирает
улики, выводит механизм, разбирает фразу как код, применяет её в бытовой сцене и получает короткое
озарение.

Проект реализован по материалам из папки [`docs/`](./docs).

## Стек

- **React 19** + **TypeScript** (strict)
- **Vite** (сборка, dev-сервер)
- **MUI** (компоненты, тёмная тема «лаборатория-расследование»)
- **React Router** (роутинг с ленивой загрузкой страниц)
- **ESLint** (flat config, type-checked) + **Prettier**
- Пакетный менеджер и рантайм — **bun**

## Дизайн

Эстетика **«Dossier forensique»** — досье французского детектива встречает терминал инженера.
Дизайн собран по принципам официального скилла Anthropic [`frontend-design`](.claude/skills/frontend-design/SKILL.md)
(борьба с «AI-slop»): вместо Inter и фиолетовых градиентов —

- **Шрифты**: Playfair Display (serif-заголовки «досье») + Manrope (тело) + JetBrains Mono (улики/код).
- **Палитра**: доминанта — янтарный «маркер улик» по тёплым чернилам, резкий красный для багов,
  функциональный sage для «решено». Использованы только осмысленно, без ровного «радужного» набора.
- **Детали**: уголки-рамки «улик», номера дел, штамп «РАСКРЫТО», корешок-папка у карточек,
  тонкая сетка-«миллиметровка» и мягкие виньетки вместо плоского фона.

Скилл `/frontend-design` доступен в проекте — его можно вызывать для дальнейших UI-задач.

## Запуск

```bash
bun install      # установить зависимости
bun run dev      # дев-сервер (http://localhost:5173)
bun run build    # прод-сборка (tsc -b + vite build)
bun run preview  # предпросмотр прод-сборки
bun run lint     # ESLint
bun run format   # Prettier
```

## Как устроено

### Сущности (см. `docs`)

**Механизм** → **Дело** (расследование) → **Улики** → **Вскрытие** (разбор как код) → **Сцена** →
**Миссия** → **Озарение**.

Дело состоит из упорядоченных шагов; каждый шаг реализует одну из **механик заданий**.

### Механики заданий

Реализовано 24 механики (все 12 ключевых из MVP + расширение; аудио-механики не реализованы):

`Найди странность` · `Выбери гипотезу` · `Разбор как код` · `Собери фразу` · `Сверни фразу` ·
`Разверни фразу` · `Исправь кальку` · `Сцена с выбором` · `Мутация фразы` · `Своя фраза` ·
`Поймай в реальности` · `Озарение` · `Найди лишнюю` · `Объясни ошибку` · `Угадай механизм (cloze)` ·
`Сортировка по корзинам` · `Уровень прямоты` · `French Debug Mode` · `Сравни две фразы` ·
`Переводческий капкан` · `Скажи проще` · `Таймлайн фон/событие` · `Диалог-квест` ·
`Найди механизмы в фразе`.

### Данные

Все задания, кейсы и данные хранятся в JSON:

```
src/data/
  categories.json          # разделы карты тем
  mechanisms.json          # механизмы (en, y, du, il faut, …)
  scenes.json              # бытовые сцены (boulangerie, médecin, …)
  cases/*.json             # дела по категориям (108 дел в 19 разделах, минимум 5 на раздел)
  index.ts                 # типизированный загрузчик данных
```

### Прогресс

Прогресс игрока (раскрытые дела, собранные озарения, собственные фразы, пойманные миссии, XP,
расписание повторения и серии «летучки») сохраняется в **localStorage** через React Context +
reducer (`src/state/`).

### Повторение («Летучка»)

Раскрытые дела «остывают» по графику интервального повторения (SM-2-lite: 1 → 3 → 7 → 21 → 60
дней). На странице **Повторение** собирается холодная перепроверка — «летучка»: смесь
перемешанных шагов раскрытых дел и пунктов «вспомни озарение» (узнать озарение механизма среди
озарений из других дел). Верный ответ двигает дело дальше по шкале и копит «крепость» механизма;
сбой возвращает дело на доследование раньше. Есть карта крепости механизмов (heatmap по тирам),
серия верных ответов и бейдж со счётчиком «к повторению» в навигации. Логика — чистые функции в
`src/lib/review.ts`.

### Перенос сейва (без бэкенда)

На странице **Настройки** (шестерёнка в шапке) прогресс экспортируется в `.json`-файл и
импортируется на другом устройстве — без аккаунта и сервера. На экспорте выбрасывается всё, что
выводится из `caseId` (тексты озарений, шаги пройденных дел), поэтому полное прохождение — это
~1 КБ. На импорте данные регидрируются из bundled-данных и валидируются: неизвестные `id` молча
отбрасываются, так что старые файлы переживают добавление новых дел. Импорт заменяет локальный
прогресс (с подтверждением). Кодек — чистые функции `exportSave`/`importSave` в
`src/lib/save-transfer.ts`.

### Структура

```
src/
  components/
    layout/                # шапка (с бейджем «к повторению»), лого, оболочка
    tasks/                 # компоненты механик заданий + TaskRenderer
    review/                # ReviewItem — карточка летучки
    CaseCard.tsx
  pages/                   # HomePage, CasePage, ReviewPage, InsightsPage, MissionsPage, SettingsPage
  state/                   # ProgressProvider, контекст, хук useProgress
  data/                    # JSON-данные + загрузчик
  lib/                     # storage, seededShuffle, review, save-transfer, useNow
  types/                   # доменная модель
  theme.ts                 # тёмная тема MUI
```
